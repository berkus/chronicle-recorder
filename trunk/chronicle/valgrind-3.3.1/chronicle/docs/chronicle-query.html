The chronicle-query Tool

chronicle-query reads a Chronicle database and DWARF2 debug information from binaries and makes the data available via a JSON query protocol on stdin/stdout.

Normally chronicle-query is spawned by a higher-level tool, such as a debugger or analysis script. It can also be spawned manually for testing.

Environment Variables

CHRONICLE_DB When set to a file name, chronicle-query will read the database from this file, unless overridden by the --db parameter.

Command-line Parameters

--db file Read the database from this file. A database must be specified, either using this parameter or the CHRONICLE_DB environment variable.

--log When specified, a file /tmp/chronicle-log.pid is created, where pid is the pid of the chronicle-query process, and all input and output is recorded in that file.
