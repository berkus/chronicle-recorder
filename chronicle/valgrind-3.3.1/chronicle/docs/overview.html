The Chronicle Tracing Infrastructure

Architecture

There are four key components:

Tracer
The tracer instruments a running program and logs memory and register changes, plus some other events, into a shared memory buffer
Indexer
The indexer process consumes the log and builds a) compressed indices that map space x time to events (reads, writes, executes) and b) a log of register change events, punctuated by full register dumps
Query Processor
Provides an asynchronous stream API for reconstructing the memory/register state of the program at any time, and also examining the indexes themselves
Application
Provides a debugging user interface which communicates with the query processor
